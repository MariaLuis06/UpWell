Perfil clinico separado da conta de email

Os dados pessoais e clinicos sao guardados noutra base de dados. A ligacao entre autenticacao e perfil e feita apenas por um token interno do servidor, nunca pelo email.

O que fica aqui

  • Idade, peso e altura para contextualizar as recomendacoes.
  • Alergias, condicoes e medicacao habitual para evitar planos perigosos.
  • Notas adicionais para limites que nao cabem nas escolhas rapidas.

Separado da conta de autenticacao.
O email fica apenas na base de autenticacao. Este perfil fica numa base separada e e identificado por um token interno indisponivel para o browser. So acessos de desenvolvimento autorizados podem consultar um perfil diretamente por token, quando `DEV_ACCESS_TOKEN` estiver configurado.
